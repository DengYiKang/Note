## Data Mining for Diagnostic Debugging in Sensor Networks: Preliminary Evidence and Lessons Learned

### 挑战

+ Non-reproducible behavior：错误无法重现。这导致传统的debug手段无法有效地进行
+ Non-local emergent behavior：单独的组件进行测试，错误可能无法体现出来，当多个组件进行合作时，它们可能会出现异常。
+ Resource constraints：资源限制这一条件使得无法用复杂的模型来检测这些异常

### 设计原则

+ 利用非确定性、不可再现性的数据：利用这些数据可以增强算法的鲁棒性，例如一个做法就是给数据集增加噪声来提高算法的泛化能力。
+ 利用数据之间的交互关系：单个数据可能不会出错，但是多个数据之间可能会发生差错，例如一段事件序列，事件的不同组合会导致不同的结果。
+ 资源问题：因为资源的限制，可以采取渐进式的诊断。

### 一些错误的例子

#### 设计上的缺陷

这个案例中涉及到多信道传感器网络MAC层协议，该协议试图利用信道多样性来提高吞吐量。该协议为每个节点分配一个主信道，将通信量很大的节点分组到同一信道上的集群中。它允许不同集群中的节点之间偶尔通信，方法是让发送方将其通道临时更改为接收方的主通道以发送消息。如果通信失败（例如，由于主信道信息过时），发送方将扫描所有信道，在新信道上寻找接收方，并相应地更新主信道信息。测试表明，总的网络吞吐量有时比单通道MAC更差。

应用于日志上的挖掘算法揭示了一个与不良日志显著相关的共同模式。该模式包括在大量节点上执行的没有接收到Ack的事件、在信道1上重试传输、在信道2上重试传输、在信道3上重试传输等。然后设计师根据这个模式发现设计上的问题。

#### 累积效应的错误

由于累积效应（如逐渐的内存泄漏或时钟溢出）而导致故障或性能问题，这种影响会在一段较长的时间内演变。

#### 竞争条件的错误

在具有高度并发性的系统中，竞争条件是导致失败的一个重要原因。
